# Operating Systems

## Process

A process is a program in execution. It contains code, data, stack, heap, and program counter.

## Thread

A thread is the smallest unit of CPU execution.

Threads within the same process share memory resources.

## Process vs Thread

Processes have separate memory spaces.

Threads share memory and resources of a process.

## Deadlock

Deadlock occurs when a group of processes wait indefinitely for resources held by one another.

Necessary conditions:

1. Mutual Exclusion
2. Hold and Wait
3. No Preemption
4. Circular Wait

## Semaphore

A semaphore is a synchronization primitive used to manage access to shared resources.

## Mutex

A mutex is a locking mechanism that ensures mutual exclusion.

## Paging

Paging divides memory into fixed-size pages and frames.

## Virtual Memory

Virtual memory allows execution of programs larger than available physical memory.

# DBMS

## Database

A database is an organized collection of related data.

## DBMS

A DBMS is software that allows users to create, manage, and manipulate databases.

## Primary Key

A primary key uniquely identifies each row in a table.

## Foreign Key

A foreign key establishes relationships between tables.

## ACID Properties

Atomicity ensures complete execution.

Consistency maintains valid states.

Isolation prevents interference between transactions.

Durability ensures permanent storage after commit.

## Normalization

Normalization reduces redundancy and improves consistency.

Normal forms:

* 1NF
* 2NF
* 3NF
* BCNF

## Joins

Types:

* Inner Join
* Left Join
* Right Join
* Full Join

## Indexing

Indexes improve query performance.

# Computer Networks

## OSI Model

Layers:

1. Physical
2. Data Link
3. Network
4. Transport
5. Session
6. Presentation
7. Application

## TCP

TCP is reliable and connection-oriented.

## UDP

UDP is connectionless and faster than TCP.

## DNS

DNS converts domain names into IP addresses.

## HTTP

HTTP is an application layer protocol for communication on the web.

## HTTPS

HTTPS uses SSL/TLS encryption for secure communication.

## Three-Way Handshake

Steps:

1. SYN
2. SYN-ACK
3. ACK

## Congestion Control

TCP uses congestion control to prevent network overload.

# Object Oriented Programming

## Class

A class is a blueprint for creating objects.

## Object

An object is an instance of a class.

## Encapsulation

Encapsulation combines data and methods into a single unit.

## Inheritance

Inheritance allows a class to acquire properties of another class.

## Polymorphism

Polymorphism allows the same interface to have different implementations.

## Abstraction

Abstraction hides implementation details.

## Constructor

A constructor initializes objects.

## Destructor

A destructor releases resources when objects are destroyed.

## Virtual Function

A virtual function enables runtime polymorphism.

# Data Structures and Algorithms

## Array

Stores elements in contiguous memory locations.

## Linked List

Stores elements using nodes connected through pointers.

## Stack

Follows LIFO.

Operations:

* Push
* Pop
* Top

## Queue

Follows FIFO.

Operations:

* Enqueue
* Dequeue

## Binary Search

Works on sorted arrays.

Time Complexity: O(log n)

## Merge Sort

Time Complexity: O(n log n)

## Quick Sort

Average Complexity: O(n log n)

Worst Case: O(n²)

## BFS

Breadth First Search explores level by level.

## DFS

Depth First Search explores depth-wise.

## Dynamic Programming

Stores results of overlapping subproblems.

## Greedy Algorithm

Makes the locally optimal choice at each step.

## Trie

Efficient data structure for string searching.

## Heap

Complete binary tree satisfying heap property.

## Segment Tree

Supports efficient range queries.

## DSU

Disjoint Set Union manages connected components.

# Aptitude

## Percentage Formula

## Profit Percentage
